# 思帆商产品矩阵升级设计方案

**日期：** 2026-06-21  
**状态：** 已确认，待实施  
**目标页面：** `/`（当前映射至 `/v1`）

## 1. 目标

将现有只有六张分类卡片、使用泛行业图库的产品矩阵，升级为可浏览、可点击、可询价的产品展示系统。新版保持思帆商现有深色科技品牌基调，同时做到清晰、简明、有展现力。

成功标准：

- 首页保留六个一级分类，用户能快速理解产品范围。
- 点击分类卡片后，在当前页面打开产品抽屉。
- 每个分类首屏展示六项；防静电服装可继续浏览完整十二款，防静电鞋类注明系列共八款。
- 点击具体产品后打开大图详情，展示产品名称、型号、参数、适用场景与询价入口。
- 桌面端和移动端均可顺畅浏览，支持键盘操作和关闭交互。
- 产品图片不再使用与产品不符的通用图库，改用统一品质、不同款式的电商产品海报。

## 2. 产品信息架构

首页继续展示六个一级分类：

1. 防静电服装：十二款，默认先展示六款，可切换查看其余六款。
2. 防静电鞋类：系列共八款，抽屉精选展示六款代表产品。
3. 无尘口罩：六款。
4. ESD 工作椅：六款。
5. 防静电地板：PVC、环氧、高架三类，每类两个方案，共六项。
6. 工程服务：六个服务方案，以工程场景海报呈现，不包装为实体商品。

产品数据统一包含：`id`、分类、中文名称、英文名称、型号、短描述、关键参数、适用场景、图片路径、标签和排序。

## 3. 视觉方向

### 3.1 总体风格

- 延续 Version A 的深海军蓝科技基调，使用电光青作为唯一主强调色。
- 减少网格、徽章和发光装饰，依靠图片、字号、留白和对比形成表现力。
- 分类卡片使用大幅产品主视觉，文字集中在底部，避免图片上散布过多信息。
- 产品海报采用统一摄影标准，但通过产品造型、配色、视角和场景差异体现不同款式。

### 3.2 电商海报规范

- 比例统一为 4:5，适配卡片和详情大图。
- 主体完整、轮廓清晰，背景干净，材质与工业用途可信。
- 图片本身不生成型号、参数或促销文字，避免乱码；所有文字由网页原生排版。
- 服装覆盖分体服、大褂、连体服、洁净服等不同结构与颜色。
- 鞋、口罩、椅子、地板和工程服务分别采用符合产品特征的构图，不重复同一模板。
- 不使用未经确认的认证标志、品牌标志或夸张性能宣称。

## 4. 页面与交互设计

### 4.1 一级产品矩阵

- 六张分类卡片保持三列布局；平板两列，手机单列。
- 每张卡片显示分类名称、英文名称、范围提示和简短规格摘要。
- 鼠标悬停时图片轻微放大，并出现“查看系列”提示。
- 整张卡片均可点击，使用真实按钮语义并提供清晰焦点状态。

### 4.2 分类产品抽屉

- 桌面端从右侧滑入，占页面约 70% 宽度；背景保留上下文并降低亮度。
- 移动端使用全屏面板。
- 顶部显示分类名称、系列说明、数量和关闭按钮。
- 主体为最多三列的六项商品网格；服装提供两组切换，共十二款。
- 点击遮罩、关闭按钮或按 `Escape` 均可关闭。

### 4.3 产品详情

- 点击产品卡打开抽屉内的详情层，避免跳转和丢失浏览位置。
- 左侧为大图，右侧为名称、型号、参数、适用场景和询价按钮。
- 移动端改为纵向排列。
- 关闭详情后返回原分类和原滚动位置。

### 4.4 询价路径

- 详情中的询价按钮复用现有联系区域或浮动联系方式。
- 点击时带入当前产品名称与型号，帮助用户明确咨询对象。
- 本阶段不新增支付、购物车、后台 CMS 或在线订单功能。

## 5. 技术结构

- 保持 Next.js 16 App Router 与现有 Tailwind CSS 4 技术栈。
- 产品数据单独放入类型化数据文件，页面组件只负责组合与展示。
- 新增客户端交互组件：产品矩阵、分类抽屉、产品详情层。
- 页面主体仍可由服务端组件输出；仅交互区域使用客户端组件。
- 图片存放在 `public/products/` 下，使用稳定的本地路径，避免远程图库失效。
- 遵循项目内 `node_modules/next/dist/docs/` 的 Next.js 16 指南，不使用已弃用的旧版 API。

## 6. 状态与数据流

用户点击分类后，产品矩阵把分类 ID 写入当前客户端状态；抽屉根据分类 ID 从本地产品数据读取列表。用户点击单品后写入产品 ID，详情层从同一数据源读取完整信息。关闭详情只清空产品 ID，关闭抽屉同时清空分类和产品 ID。

所有内容均为本地静态数据，不需要加载态。若图片加载失败，卡片保留产品名称、型号和纯色背景；若产品 ID 无效，不打开详情层。

## 7. 可访问性与响应式

- 所有可点击卡片均使用按钮或链接语义。
- 抽屉和详情层使用对话框语义、可读标题和焦点管理。
- 打开弹层后锁定页面滚动，关闭后恢复焦点。
- 支持 `Escape` 关闭，并提供不依赖悬停的移动端入口。
- 图片提供准确的中文替代文本。
- 动画遵循 `prefers-reduced-motion`，减少动态偏好的用户不播放位移动画。

## 8. 验证标准

- 运行 lint 与生产构建，确保没有 TypeScript、ESLint 或 Next.js 构建错误。
- 浏览器验证六个分类均可打开，服装十二款可完整浏览，每个分类至少六项可见。
- 逐类验证单品详情、关闭逻辑、询价入口与焦点恢复。
- 在桌面、平板和手机尺寸检查图片裁切、文字换行、抽屉宽度和详情布局。
- 检查生成图片与产品名称一致，无错误文字、明显畸变、重复构图或不相关主体。
- 最终对照设计概念图与浏览器截图，确认视觉层级和实现一致。

## 9. 实施边界

本次只升级当前默认首页 `/v1` 对应的产品矩阵及其交互，并补充所需本地产品素材。`/v2` 与 `/v3` 保持现状，不新增产品详情路由、CMS、购物车或订单系统。
